- Use GitHub-flavored Markdown for documentation.

## Repository Structure

- `src/` or `app/`: Main source code
- `tests/` or `spec/`: Test files
- `docs/`: Documentation
- `tasks/`: Task-related files and documentation
- `.github/`: GitHub workflows and templates
